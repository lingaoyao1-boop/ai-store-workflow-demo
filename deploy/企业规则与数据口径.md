# 企业规则与数据口径

## 业务目标

在避免缺货的同时，控制鲜果、奶制品和甜品原料的临期损耗。系统输出的是“建议补货量”，最终订单仍由店长确认。

## 硬性规则

1. 预测周期：未来3天。
2. 安全库存：额外保留1.5天预测销量。
3. 最大库存：库存覆盖不得超过7天，超过时停止补货。
4. 包装规格：补货量按5个单位向上取整。
5. 单次补货上限：每个商品最多180个单位，防止模型或异常数据造成大额订单。
6. 人工审批：单品建议补货达到100个单位时必须人工确认。
7. 临期预警：最近批次距离到期3天及以内时，检查现有库存能否在保质期内售完。
8. 异常销量：近7日均销比前7日增长35%以上时标记异常，要求确认促销、天气或活动影响。
9. AI权限：DeepSeek只能解释规则结果、生成行动建议，不得修改补货数字。

## 计算口径

- 7日均销 = 最近7天销量 / 7。
- 趋势系数 = 1 + 最近7日相对前7日的增长率，限制在0.85至1.35之间。
- 日预测销量 = 7日均销 × 趋势系数。
- 可用库存 = 当前库存 + 在途库存。
- 库存覆盖天数 = 可用库存 / 日预测销量。
- 目标库存 = 日预测销量 ×（3天预测周期 + 1.5天安全库存）。
- 建议补货量 = 目标库存 - 可用库存，再按包装规格取整并应用上下限规则。

## 演示数据说明

- 共8款饮品，覆盖鲜果、奶茶、咖啡和甜品。
- 每款商品包含30天历史销量、当前库存、在途库存和最近批次到期天数。
- 数据根据连锁饮品门店的常见销量波动人工模拟，只用于作品集和流程演示，不代表卡旺卡真实经营数据。

## 为什么需要DeepSeek

规则引擎适合确定性计算，但店长不希望阅读大量数字。DeepSeek将精简后的异常SKU和规则结果转换为：

1. 今日经营结论。
2. 缺货、临期和异常增长风险清单。
3. 店长可执行的补货、调拨、促销或盘点动作。
4. 需要人工确认的事项。

因此，大模型承担“解释和沟通”，规则引擎承担“计算和约束”。
